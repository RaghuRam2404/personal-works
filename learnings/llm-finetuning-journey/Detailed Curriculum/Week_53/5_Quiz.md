# Week 53 Quiz — Data Quality Strategy

**Difficulty:** Senior research engineer. Every question requires reasoning, not recall.

---

## Multiple Choice

**Q1.** The LIMA paper trains on 1,000 examples and matches models trained on 50,000+. Which statement best explains why?

A) Pre-training already encodes most of the knowledge; alignment fine-tuning only adjusts output style and format.
B) 1,000 examples trigger less catastrophic forgetting than 50,000 examples.
C) The 50,000-example models use a lower learning rate, causing underfitting.
D) LIMA uses a proprietary data format that is more token-efficient.

---

**Q2.** You run MinHash deduplication on your v2 dataset (18K examples, threshold 0.85) and find 6,000 near-duplicate pairs. What is the correct next step?

A) Delete all examples involved in near-duplicate pairs — both copies of each pair.
B) From each near-duplicate cluster, keep the example with the most complex SQL (highest AST depth).
C) Keep deduplication as-is; near-duplicates provide useful augmentation during training.
D) Lower the threshold to 0.70 to catch more duplicates.

---

**Q3.** Tulu 3 removes examples from training data that appear in known benchmark test sets. Your Spider-augmented examples were generated by paraphrasing Spider training set questions. Should you remove them?

A) Yes — any Spider-derived example is contaminated.
B) No — only Spider test set contamination matters; training set examples are fine.
C) It depends on whether your eval uses Spider test set; if yes, only test set overlap matters.
D) Yes — even training set overlap inflates results because the model memorizes phrasings.

---

**Q4.** Your v3 data card targets 50K examples. You currently have 18K in v2 after deduplication. You plan to generate 30K synthetic examples via a teacher model. Which risk is most critical to address before starting generation?

A) The teacher model may generate SQL in MySQL dialect rather than PostgreSQL dialect.
B) 30K examples will take too long to generate.
C) The synthetic examples may not cover your hardest TimescaleDB skills if teacher prompts are generic.
D) HuggingFace datasets has a 50K example limit.

## Deep Scenario

**Q8.** You have a budget to generate 30K synthetic SQL examples using GPT-4o as teacher. Your skill taxonomy has 15 skill categories, but you can only afford prompts for 30K total examples. Design a concrete allocation strategy:

- How many examples per skill?
- How do you handle skills where the teacher model is known to perform poorly (e.g., TimescaleDB hyperfunctions, which appeared rarely in GPT-4o's pretraining data)?
- How does difficulty stratification interact with the allocation?
- What is your fallback if the teacher produces < 60% execution-correct SQL for a particular skill?

Write a structured response with specific numbers and decisions.
