# Week 53 Glossary

**LIMA (Less Is More for Alignment):** A 2023 paper showing that 1,000 high-quality diverse examples can match models trained on far more data for alignment fine-tuning.

**Tulu 3:** AllenAI's fully open post-training recipe (2024) covering data curation, SFT, DPO, and RLVR for instruction-following models.

**Skill taxonomy:** An explicit enumeration of the capabilities a dataset should teach, used to ensure balanced coverage and prevent any skill being crowded out.

**Near-duplicate:** Two training examples with Jaccard similarity > 0.85 on their 5-gram sets; effectively redundant and harmful for generalization.

**MinHash:** A probabilistic data structure for efficiently estimating set similarity (Jaccard index) without pairwise comparison; enables O(n) deduplication.

**LSH (Locality-Sensitive Hashing):** A technique that groups similar items into the same hash bucket, enabling approximate nearest-neighbor search at scale; used with MinHash for deduplication.

**Benchmark contamination:** When training data contains examples from a benchmark's test set, causing artificially inflated evaluation scores.

**Data card:** A structured document specifying a dataset's intended use, composition, quality criteria, and known limitations; the dataset equivalent of a model card.

**Execution correctness:** A SQL quality criterion requiring that the generated query runs without error against the target database.

**Semantic correctness:** A SQL quality criterion requiring that the query returns the logically correct answer to the natural language question, not just any valid result set.

**Difficulty stratification:** Deliberately dividing a dataset into Easy/Medium/Hard/Expert tiers to ensure the model trains on a range of query complexity levels.

**Off-policy data:** Training examples generated by a model (teacher) whose behavior differs significantly from the student model being trained; can cause instability if style gap is too large.

**AST depth:** The depth of a SQL query's abstract syntax tree, used as a proxy for query complexity; deeper ASTs indicate more nested or complex queries.

**RLAIF (Reinforcement Learning from AI Feedback):** Using a stronger AI model to generate preference signals for training, instead of human annotators.

**Schema diversity:** The number of distinct database schemas (table/column structures) represented in a training dataset; low diversity causes memorization over generalization.
