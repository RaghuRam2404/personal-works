# Week 72 Glossary — Frontier Reading: DeepSeek and Qwen

**Mixture of Experts (MoE)**: An LLM architecture where each transformer layer contains multiple FFN "expert" sub-networks; a learned router selects K active experts per token, reducing active FLOPs while scaling total parameters.

**Expert routing**: The learned function (typically a linear projection + softmax + Top-K) that selects which MoE expert sub-networks process each token.

**Load balancing (MoE)**: The mechanism that ensures routing is distributed across experts rather than all tokens going to a few experts; DeepSeek-V3 uses a bias-based approach instead of an auxiliary loss.

**Multi-head Latent Attention (MLA)**: DeepSeek's attention variant that compresses K and V into a low-dimensional latent vector before caching, reducing KV cache memory by 5–10x.

**KV cache compression**: Reducing the memory required to store key-value pairs for attention in cached contexts; enables longer contexts at fixed VRAM budget.

**DeepSeek-R1**: A 671B reasoning model trained entirely through RL with verifiable rewards; then distilled into smaller models via rejection-sampling supervised fine-tuning.

**Cold start (R1)**: A small-scale SFT step on curated chain-of-thought demonstrations before GRPO, needed to initialize coherent reasoning behavior in the base model.

**Rejection sampling**: Generating many completions, keeping only those that pass a correctness check (execution, math equality), and using the passing examples as SFT training data.

**Knowledge distillation (reasoning)**: Training a small model on the successful reasoning traces generated by a larger model; transfers reasoning capability without requiring the small model to run RL.

**Qwen2.5-Coder**: The code-specialized variant of Qwen2.5 trained with code-focused SFT and RLHF on top of the base 18T-token pretrained model; your current base model.
