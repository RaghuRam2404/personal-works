# Week 16 Glossary

**Phase Gate**: A structured checkpoint at the end of a phase; must be passed (all criteria met) before proceeding to the next phase.

**Self-assessment**: Honest evaluation of your own knowledge gaps against stated criteria; required for effective learning.

**From-memory implementation**: Writing code without any external reference; the gold standard for verifying true understanding vs. pattern-matching.

**Capstone project**: An integrative implementation that combines all skills from a phase; for Phase 2 this is a from-scratch modern transformer trained on SQL.

**Knowledge map**: A structured view of what each week contributes to the overall learning objective; used in Week 16 to review the phase.

**Go/No-Go decision**: A binary decision point: proceed to Phase 3 (go) or repeat failing weeks (no-go); should be made honestly based on gate project and self-assessment results.

**Diagnostic habit**: The practice of formulating hypotheses about failure modes and checking them systematically; essential for debugging training runs and production models.

**Conceptual mastery vs. implementation ability**: Knowing what something does vs. being able to implement it from scratch; Phase 2's gate tests the latter.
