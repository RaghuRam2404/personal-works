# Week 59 Glossary

**DPO (Direct Preference Optimization):** An alignment method that directly optimizes a language model on preference pairs (chosen vs. rejected) without a separate reward model; derived from the RLHF objective through a closed-form policy transformation.

**Preference pair:** A training example for DPO consisting of a prompt, a "chosen" (preferred) response, and a "rejected" (dispreferred) response.

**Beta (β):** The DPO temperature parameter controlling the KL-divergence constraint strength; lower β = more aggressive preference optimization, higher β = more conservative.

**Reference model (π_ref):** The frozen model used in DPO's KL constraint; typically the SFT model checkpoint that precedes DPO training.

**Reward margin:** The difference between the implicit rewards assigned to chosen and rejected responses; `E[reward(chosen)] - E[reward(rejected)]`; should be positive and increasing during DPO training.

**On-policy data:** Preference pairs where both chosen and rejected responses were generated by the model being trained (or a close approximation of it); provides better-calibrated DPO updates than off-policy data.

**Off-policy data:** Preference pairs generated by a different model (e.g., teacher model chosen vs. Phase 5 model rejected); less efficient for DPO because the KL constraint is calibrated to the wrong reference distribution.

**DPO loss going negative:** A situation where the DPO loss falls significantly below zero, indicating the model has over-concentrated on chosen responses and may have collapsed its distribution; requires early stopping or higher beta.

**Hard negative:** A rejected response that is plausible and non-trivially wrong; provides stronger learning signal than easy negatives (obvious errors); for SQL, "executes correctly but returns wrong rows" is a hard negative.

**Distribution collapse:** A failure mode in alignment training where the model becomes overconfident and generates only a narrow set of outputs, losing the diversity of the SFT model.

**LoRA reference trick:** A TRL optimization where `ref_model=None` is passed to DPOTrainer; TRL computes reference log-probabilities by temporarily disabling the LoRA adapters, saving ~14GB VRAM versus storing a separate reference model copy.
