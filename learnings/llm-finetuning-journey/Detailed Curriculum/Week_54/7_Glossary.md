# Week 54 Glossary

**Self-Instruct:** A method for bootstrapping instruction-following data by prompting an LLM to generate new tasks from a small seed set of human-written examples.

**Magpie:** A 2024 data generation approach that exploits a model's chat template — feeding only the system prompt and partial user turn, letting the model complete both the instruction and response self-consistently.

**Genie:** A 2024 grounded synthetic data generation approach that conditions the teacher model on specific documents or schemas to prevent hallucinated content.

**Teacher model:** A strong model (e.g., GPT-4o) used to generate training data for a weaker student model (e.g., your 7B Qwen); the teacher is not fine-tuned, only prompted.

**Schema grounding:** Including the target database DDL (CREATE TABLE statements) in the teacher prompt to prevent hallucinated column/table names.

**Schema hallucination:** When a teacher model generates SQL referencing columns or tables that do not exist in the target schema; the primary failure mode of ungrounded SQL generation.

**Execution validation:** Running generated SQL against a live PostgreSQL instance (optionally within a transaction that is rolled back) to verify the query does not error.

**Parse rate:** The fraction of teacher API responses that successfully parse into structured output (JSON); a proxy for prompt quality and output format clarity.

**Execution rate:** The fraction of generated SQL queries that execute without error against the target database; the primary quality signal in SQL data generation.

**Exponential backoff:** A retry strategy where the wait time between retries doubles with each attempt (1s, 2s, 4s, 8s...); used to handle API rate limit (429) errors gracefully.

**AsyncIO Semaphore:** A Python primitive that limits the number of concurrent coroutines; used to cap concurrent API calls and avoid rate limit errors.

**Checkpoint (training pipeline):** A periodically saved state of a running process, enabling recovery from crashes without restarting from scratch.

**Diversity pressure:** A prompt instruction that provides recent generated examples and asks the teacher to avoid producing similar outputs; prevents clustering around common phrasings.

**On-policy data:** Training examples generated by (or stylistically consistent with) the student model being trained; generally more stable for fine-tuning than strongly off-policy data.
