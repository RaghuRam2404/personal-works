# Week 41 Glossary

**MDP (Markov Decision Process)**: A 5-tuple (S, A, P, R, γ) defining a sequential decision-making problem with states, actions, transitions, rewards, and discount.

**Markov property**: The next state depends only on the current state and action, not on history.

**Policy π(a|s)**: A probability distribution over actions given a state; for an LLM, the softmax output.

**Value function V^π(s)**: Expected cumulative discounted return from state s under policy π.

**Q-function Q^π(s,a)**: Expected cumulative discounted return from state s taking action a, then following π.

**Advantage function A(s,a)**: Q(s,a) minus V(s); measures how much better action a is than average.

**REINFORCE**: Simplest policy gradient algorithm; collects full episodes then updates using ∇_θ log π · G_t.

**Policy gradient theorem**: States that ∇_θ J(θ) = E[Σ_t ∇_θ log π_θ(a_t|s_t) · G_t] without needing to differentiate the environment.

**Log-derivative trick**: Identity ∇_θ p(τ|θ) = p(τ|θ) · ∇_θ log p(τ|θ); enables Monte Carlo gradient estimation.

**Return G_t**: Cumulative discounted reward from time step t onward: Σ_{k≥t} γ^(k-t) r_k.

**Discount factor γ**: Scalar in [0,1) that downweights future rewards; γ=0.99 is common for episodic tasks.

**On-policy**: Learning method that uses data generated by the current policy for each update.

**Off-policy**: Learning method that can reuse experience from older or different policies (e.g., via replay buffer).

**Sparse reward**: Reward signal received only at episode termination, not at each step.

**Baseline (in policy gradient)**: State-dependent scalar b(s_t) subtracted from returns to reduce gradient variance without introducing bias.

**Episode**: One complete trajectory from initial state to terminal state; in LLMs, one full generation.
