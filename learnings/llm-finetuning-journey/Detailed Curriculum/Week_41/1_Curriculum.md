## Concepts

### Markov Decision Processes

An MDP is a 5-tuple: (S, A, P, R, γ).

- S: state space
- A: action space
- P(s' | s, a): transition probability
- R(s, a): reward function
- γ ∈ [0, 1): discount factor

The Markov property says the next state depends only on the current state and action, not on history. In an LLM context: the state is the current token sequence (prompt + tokens generated so far), the action is the next token to generate, the transition is deterministic (given token t, the sequence grows by one token), and the reward is typically 0 for all intermediate tokens and a scalar from a reward model at the end-of-sequence token. This is a sparse reward problem.

### Policy and Value Functions

A policy π(a|s) is a probability distribution over actions given a state. For an LLM, this is precisely the softmax output of the model: π_θ(token | context).

The value function V^π(s) is the expected cumulative discounted return when starting from state s and following policy π:

```
V^π(s) = E_π [ Σ_{t=0}^{∞} γ^t R(s_t, a_t) | s_0 = s ]
```

The Q-function (action-value) Q^π(s, a) is the same but also conditions on the first action:

```
Q^π(s, a) = E_π [ Σ_{t=0}^{∞} γ^t R(s_t, a_t) | s_0 = s, a_0 = a ]
```

The advantage function A^π(s, a) = Q^π(s, a) − V^π(s) answers: "how much better is action a than the average action in state s?" This will be central to PPO next week.

### On-Policy vs. Off-Policy

On-policy methods (REINFORCE, PPO) use data generated by the current policy to update that same policy. You cannot reuse old experience without correcting for the distribution shift. Off-policy methods (DQN, SAC) can use a replay buffer of old experience and apply importance sampling corrections.

For LLMs: RLHF/PPO is on-policy — the model generates completions, a reward model scores them, and the model updates. GRPO (Week 46) is also on-policy. DPO (Week 43) is technically off-policy in that it trains on a fixed preference dataset, not on completions from the current model.

### The Policy Gradient Theorem

The key result: the gradient of expected return J(θ) = E_{τ∼π_θ}[R(τ)] with respect to policy parameters θ is:

```
∇_θ J(θ) = E_{τ∼π_θ} [ Σ_t ∇_θ log π_θ(a_t | s_t) · G_t ]
```

where G_t = Σ_{k=t}^{T} γ^{k-t} r_k is the return from step t onward.

The key insight: you do not need to differentiate through the environment (which may be non-differentiable or unknown). You only need to differentiate through your own policy log-probabilities. This is why the theorem is so powerful — and it is why you can apply it to LLMs even though "executing SQL against Postgres" is a non-differentiable reward.

### REINFORCE Algorithm

REINFORCE is the simplest policy gradient algorithm:

1. Sample a complete trajectory (episode) from π_θ
2. Compute returns G_t at each step
3. Compute gradient: ∇_θ J ≈ Σ_t G_t · ∇_θ log π_θ(a_t | s_t)
4. Update: θ ← θ + α · ∇_θ J

Problems with vanilla REINFORCE:
- High variance: the return G_t includes randomness from all future steps
- Sample inefficiency: you discard each trajectory after one update
- Sparse reward makes the signal noisy: if reward only comes at the end, early tokens get a diluted learning signal

Baseline subtraction: replacing G_t with (G_t − b) where b is a baseline (often V^π(s_t)) reduces variance without introducing bias. This is the origin of the advantage function A_t = G_t − V(s_t), which PPO uses.

### Mapping to LLMs

| RL concept | LLM equivalent |
|---|---|
| State s_t | (prompt + tokens generated so far) |
| Action a_t | Next token from vocabulary |
| Policy π_θ | LLM softmax output |
| Episode | One complete generation (prompt → EOS) |
| Reward R | Reward model score at EOS |
| Return G_0 | Total reward (usually just R since sparse) |

The episode length for a language model is the number of generated tokens, which can be 50–2000. This makes the credit assignment problem severe: which of the 500 tokens deserved the positive reward?

## Common Misconceptions

- "RL only applies to games." RL is a mathematical framework for sequential decision making. Language generation is sequential decision making.
- "The policy gradient requires a differentiable reward." It does not. The log π term is differentiable; the reward R acts as a scalar weight. This is the whole point.
- "Off-policy is always better because it reuses data." Not necessarily — for LLMs, on-policy is often preferred because the distribution of generated text shifts rapidly as the model updates.
- "REINFORCE is what RLHF uses." No. RLHF uses PPO, which addresses REINFORCE's high-variance and sample-inefficiency problems. REINFORCE is the conceptual foundation.
- "Discount factor γ = 1 is fine for language." At γ = 1 with long episodes, returns blow up. In LLM RLHF, the KL penalty acts as an implicit regularizer instead.
