# Week 44 Glossary

**RLAIF (Reinforcement Learning from AI Feedback)**: Alignment pipeline where a strong AI model (not humans) labels preferences or provides reward signals.

**Constitutional AI (CAI)**: Anthropic's framework for defining written principles ("constitutions") that an AI uses to evaluate and revise its own outputs.

**Execution-based preference labeling**: Labeling SQL preference pairs by running both candidates on a real database and designating the one that executes correctly as "chosen".

**Verifiable reward**: A reward signal that can be computed exactly by a deterministic process (e.g., SQL execution, unit test passing), without human or AI judgment.

**Statement timeout**: A Postgres setting (SET statement_timeout = N) that kills any query running longer than N milliseconds; prevents infinite loops from hanging the labeling pipeline.

**Prompt set**: The collection of natural language questions used to generate SQL candidates; should cover the distribution of queries the deployed model will receive.

**Reference SQL**: A known-correct SQL query for a given prompt, used to compare row-level outputs for labeling execution-correct but semantically wrong pairs.

**Discard rate**: The fraction of candidate pairs discarded during labeling (both execute or both fail); a high discard rate indicates the prompt set mismatches the schema.

**DML (Data Manipulation Language)**: SQL statements that modify data (INSERT, UPDATE, DELETE, DROP); must be blocked in the execution harness to prevent data corruption during labeling.

**Preference pair**: A tuple of (prompt, chosen completion, rejected completion) used to train DPO or reward models.

**Schema hallucination**: When a model generates SQL referencing table or column names that do not exist in the actual database schema.

**Within-model pairs**: Preference pairs generated by running the same model twice with different temperatures; useful for capturing stochastic diversity but often lower signal than cross-model pairs.
